# RoundFi ÔÇö Project Status

**Updated:** 2026-05-07 ┬Ě **Tracking branch:** `main` ┬Ě **Live:** [roundfinancial.vercel.app](https://roundfinancial.vercel.app) ┬Ě **Devnet:** [4 programs ┬Ě protocol initialized ┬Ě pool fully completed end-to-end (3 cycles, 9 contributions, 3 payouts, status = Completed)](./devnet-deployment.md)

> **TL;DR for evaluators / partners.** RoundFi is a multi-track project: a polished **off-chain product** (front-end + L1 actuarial simulator) is live and demoable today, and the **on-chain Anchor programs are deployed, initialized, AND have closed a full ROSCA pool end-to-end on Solana Devnet, with two Triple Shield guards captured firing on real funds** ÔÇö all 3 members received their $30 USDC credit in their respective slot, all 3 cycles closed, `Pool.status = Completed`. The protocol now carries: `ProtocolConfig` + `ReputationConfig` + `Pool` PDAs (4 USDC vault ATAs: ~$67.50 in escrow + ~$0.90 in solidarity; pool float drained to ~0 by 3 ├Ś $30 credit transfers) + 3 `Member` PDAs (Metaplex Core position NFTs minted; all `paid_out=true`, each ATA holds the +$30 credit they received) + **12 reputation `Attestation` PDAs** (9 ├Ś `SCHEMA_LATE` from cycle contributions + 3 ├Ś `SCHEMA_CYCLE_COMPLETE` from the slot payouts). **Plus two on-chain enforcement events**: `WaterfallUnderflow` would have refused `claim_payout` while pool float < credit (top-up flow proves the guard), and `release_escrow` reverts on-chain with `EscrowLocked` (code 6011) when `on_time_count < checkpoint` (durable failed-tx evidence). All verifiable via Solscan ([deployment register](./devnet-deployment.md)). `anchor build` is a required CI gate on `main`; the full bankrun integration suite is the next layer of verification. The B2B oracle and production indexer are explicitly **roadmap (Phase 3 / post-grant)**. This doc is the authoritative status register ÔÇö anything claimed elsewhere should reduce to one of the three columns below.
